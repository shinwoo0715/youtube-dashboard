📊 유튜브 채널 완전 분석 프로그램 개발 요청 프롬프트
1. 프로그램 개요
사용자가 유튜브 채널명 또는 URL을 입력하면,

해당 채널의 롱폼/쇼츠 영상 전체 데이터를 수집·분석하고,

진행상황을 실시간 표시, 에러/경고 안내,

모바일에서도 완전히 최적화된 반응형 UI로 결과를 보여주는

웹앱 또는 데스크탑 앱을 만들어 주세요.

(OpenAI, ChatGPT 등 AI 사용 없이) Python(및 pandas, matplotlib 등)만으로
모든 데이터 분석·시각화를 구현해야 합니다.

2. 입력/파싱(채널명/URL 인식) 기능
사용자는 입력란에 유튜브 채널명 또는 다양한 형태의 유튜브 채널 URL을 입력할 수 있어야 합니다.

입력 예시:

칙칙풉풉

https://www.youtube.com/@칙칙풉풉

https://www.youtube.com/@%EC%B9%99%EC%B9%99%ED%92%89%ED%92%89

https://www.youtube.com/@%EC%B9%99%EC%B9%99%ED%92%89%ED%92%89/shorts

https://www.youtube.com/channel/UCxxx

https://www.youtube.com/user/아이디

(한글, 영어, 퍼센트 인코딩, @, /shorts 등 URL 내 어떤 부가 요소가 와도)

입력값에서 채널핸들(@username), 채널ID, 채널명을 자동 추출/파싱해서,
어떤 경우에도 올바른 채널 데이터를 가져와야 합니다.

한글 퍼센트 인코딩 URL(예: @%EC%B9...)도 문제없이 인식해야 하고,
/shorts, /videos, /featured 등 뒤에 뭐가 오든 정확하게 작동해야 합니다.

“채널명/URL 자동 인식 파서”는 독립 함수로 설계하고,
실제 API로 해당 채널의 정확한 ID를 찾아야 함.

3. API키 입력 및 보안
YouTube Data API Key는
반드시 사용자별로 안전하게 입력할 수 있게
입력창(패스워드 모드 지원) 제공

API키를 입력하지 않으면 진행 불가하도록 체크 및 안내

(API키는 코드에 하드코딩하지 않음. 서버/프론트 분리시 백엔드에서만 처리)

4. 진행상황 표시/에러 안내
데이터 수집분석시각화~보고서 작성 등 전체 단계별로
진행상황을 UI에서 실시간 표시 (예: “채널 정보 수집 중…”, “영상 데이터 분석 중…”, “차트 생성 완료!” 등)

오류/예외 발생 시 명확한 안내 메시지(무슨 문제가 발생했고, 사용자가 어떻게 하면 되는지까지 설명)

(예: API키 오류, 채널 없음, API 제한 초과, 영상 없음 등)

5. 모바일 최적화 UI(반응형 웹)
모든 기능/결과가 모바일/PC에서 완전히 보기 편한 UI여야 함

입력란, 버튼, 테이블, 차트 등 손가락 터치/스크롤 위주로 편리하게

가로/세로, 화면 크기 변화에도 자연스러운 자동 배치

데이터 테이블/차트는 모바일에서 가로 스크롤/확대 축소/접힘 가능 등 유연하게

(Streamlit, Dash, Plotly Dash, React+Flask, React+FastAPI, 또는 모바일 대응 데스크탑 전자앱 등 모두 OK)

6. 데이터 수집 범위
입력한 채널의 모든 공개 영상(롱폼/쇼츠 모두, 최대 1000개 이상 지원이면 더 좋음)

각 영상별 데이터:

제목, 업로드 날짜/시간, 조회수, 좋아요수, 댓글수, 영상 길이(초/분), 유형(쇼츠/롱폼 구분),

태그, 설명, 썸네일,

(가능하다면) 자막 자동 수집

영상 유형 구분:

60초 이하면 “쇼츠”, 그 외 “롱폼”(or #shorts 해시태그 등도 참고)

7. 데이터 분석/시각화(최중요)
데이터 시각화/분석에 최우선 집중!

1) 업로드 추이/패턴

월별, 요일별, 시간대별 업로드 횟수/평균 조회수/좋아요/댓글 통계(롱폼/쇼츠 구분)

2) 상위 영상/터진 영상 분석

조회수/좋아요/댓글 Top10~20 영상 추출 및 표/차트

“터진 영상”들의 특징(제목 단어, 길이, 업로드 요일/시간, 태그 등)

3) 키워드·해시태그·자주 쓰는 단어 분석

제목/설명/태그에서 자주 등장한 단어(불용어 제거, 빈도수 카운트)

워드클라우드, TOP20 키워드 빈도 그래프

상위 영상의 키워드/제목 패턴 따로 분석(“터진 영상들은 어떤 단어/구성이 많은가?”)

4) 표/리스트/필터링

영상 데이터 전체 표로(정렬, 필터, 검색)

특정 조건(예: 쇼츠만, 조회수 1만 이상만 등) 필터링/다운로드 가능

5) 기타

영상 길이별 조회수/참여율 등 상관관계 분석

업로드 시간/요일별 평균/최고 조회수 분석 등

8. 결과 리포트(요약)
데이터/차트/인사이트가
한눈에 보이게 대시보드(요약 화면)와,
세부 분석 표/차트(탭/섹션 분리)

주요 차트/표/워드클라우드 등 다운로드/이미지 저장 지원

“이 채널이 잘 되는 이유/특징”은
AI 없이 통계적 근거(상위 영상 패턴, 키워드 등) 기반으로 자동 요약

결과 데이터/차트/표 엑셀/CSV로 내보내기 지원이면 더 좋음

9. 기술 조건/제한
OpenAI, ChatGPT, Claude 등 외부 AI 서비스는 절대 사용하지 않음

모든 요약/분석/차트는 Python만으로(NumPy, pandas, matplotlib, seaborn, plotly, wordcloud 등)

YouTube API 사용(크롤링 금지, 공식 API만 사용)

소스코드 내 “입력/파싱/분석/시각화” 부분은 함수별로 최대한 명확하게 분리

주석/코드 설명 충분히 포함



어떤 키워드를 많이썼고 몇번썼고 상위의 영상들은 이런 키워드들을 많이 썼고 이런것들(제목부분)
데이터수집은 그냥 다 가져와 다 가져올 수 있는거 가능한것들
